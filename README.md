# Machine Learning Portfolio — Sarosh Kumar

A modern, single-page portfolio documenting my machine learning journey through the course,
built around a complete flagship project: an **HR Employee Attrition Predictor** on the
IBM HR Analytics dataset.

🔗 **Live site:** enable GitHub Pages (Settings → Pages → Deploy from branch → `main` / root).

## What's inside

- **The Journey** — data understanding → EDA → preprocessing → modelling → evaluation → interpretation.
- **Flagship Project** — an end-to-end Python ML system that predicts whether an employee will
  leave, explains why, and recommends retention actions.
- **What I Learned** — every applied concept: scaling & normalization, Logistic Regression,
  Decision Trees, Random Forest, **Bagging**, **Boosting**, **Stacking**, confusion matrices,
  F1 / ROC-AUC, threshold tuning for imbalance, and cross-validation.
- **Results** — six models compared honestly, with the accuracy-vs-F1 trade-off made explicit.

## Tech

Plain, dependency-free **HTML + CSS + JavaScript** — fast, portable, and easy to host on GitHub Pages.

```
index.html      # structure & content
styles.css      # modern dark theme, responsive
script.js       # scroll reveals + active nav
assets/         # real charts generated by the project
```

## The project behind it

| Model | Accuracy | F1 | ROC-AUC |
|-------|----------|-----|---------|
| Stacking Ensemble | 0.884 | 0.575 | 0.820 |
| Logistic Regression | 0.874 | 0.575 | 0.811 |
| Gradient Boosting | 0.704 | 0.466 | 0.814 |
| Random Forest | 0.806 | 0.496 | 0.800 |
| Bagging | 0.772 | 0.481 | 0.784 |
| Decision Tree | 0.694 | 0.357 | 0.696 |

> Evaluated at an imbalance-adjusted 0.40 decision threshold. On a dataset that is ~84% "stay",
> raw accuracy is a weak metric — F1, recall and ROC-AUC are what demonstrate real value.
